## 2. Logic Chain

From the above observations, we derive the exact parameters for sitemap generation:

1. **Total Page Calculation**:
   - We have 17 static routes and 28 geo city routes.
   - Total base paths = 17 + 28 = 45 base paths.
   - We have 3 active locales (`de`, `en`, `ar`).
   - Every localized path represents a distinct, crawlable URL.
   - 45 base paths * 3 locales = **135 sitemap pages in total**.
   - This aligns precisely with the user request's requirement for 135 total pages.

2. **Multilingual Alternates Structure**:
   - Next.js 15 supports the `alternates` property on sitemap entries.
   - Each entry in the sitemap return array must specify `url` and `alternates.languages` listing the equivalents for `de`, `en`, `ar` along with `x-default` (pointing to `de`).
   - For homepage routes (empty path), the output URLs must end with a trailing slash (`/`), whereas subpages must not, in order to comply with the rules enforced in `tests/seo_adversarial.spec.ts`.

3. **Robots.txt Configuration**:
   - Next.js 15 generates `/robots.txt` from `app/robots.ts` by returning a `MetadataRoute.Robots` object.
   - The robots configuration must reference the dynamic `/sitemap.xml` generated by Next.js using the host URL set in the environment (`NEXT_PUBLIC_SITE_URL`).
   - The `host` key in the robots return object must be explicitly populated with the site URL, which maps to the Yandex-compatible `Host` directive in the generated `robots.txt`.

## 3. Caveats

- **Datenschutz/Privacy**: Although a link to `/datenschutz` is present in the footer and is translated, the route directory `app/[locale]/datenschutz` does not exist in the source code. It has been excluded from the active routes list in the proposed sitemap to avoid generating broken 404 links, which would harm search engine crawl efficiency.
- **Dynamic Site URL**: The sitemap and robots generation depends on the `NEXT_PUBLIC_SITE_URL` environment variable. If not set, it defaults to `https://k-aqua.de`.
- **Failing Stress Test**: The pre-existing math bug in `haversineKm` does not affect sitemap or robots generation since distance calculations are not required for SEO configuration, but it remains present in the main test suite.

## 4. Conclusion & Proposed Implementation Code Structures

To complete Step 20's sitemap and robots objectives, the following files should be created:

### Proposed `app/sitemap.ts`

```ts
import type { MetadataRoute } from 'next';
import { GEO_MARKETS } from '@/lib/data/geo';
import { routing } from '@/lib/i18n/routing';

export const dynamic = 'force-static';

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || 'https://k-aqua.de';
  
  // 17 core static routes (without dynamic segments)
  const coreRoutes = [
    '',
    'produkte',
    'produkte/finder',
    'loesungen',
    'co2-rechner',
    'academy',
    'trust-center',
    'partnerschaft',
    'service',
    'maerkte',
    'referenzen',
    'unternehmen',
    'karriere',
    'projektanfrage',
    'news',
    'kontakt',
    'impressum',
  ];

  // Helper to format the localized URL properly matching SEO constraints
  const getLocalizedUrl = (locale: string, path: string) => {
    if (!path) {
      return `${siteUrl}/${locale}/`;
    }
    return `${siteUrl}/${locale}/${path}`;
  };

  const sitemapEntries: MetadataRoute.Sitemap = [];

  // Generate sitemap entries for core static routes
  for (const path of coreRoutes) {
    for (const locale of routing.locales) {
      sitemapEntries.push({
        url: getLocalizedUrl(locale, path),
        lastModified: new Date('2026-06-14T15:00:00Z'), // static build date for deterministic outputs
        changeFrequency: 'weekly',
        priority: path === '' ? 1.0 : 0.8,
        alternates: {
          languages: {
            de: getLocalizedUrl('de', path),
            en: getLocalizedUrl('en', path),
            ar: getLocalizedUrl('ar', path),
            'x-default': getLocalizedUrl('de', path),
          },
        },
      });
    }
  }

  // Generate sitemap entries for programmatic geo-city routes (28 markets)
  for (const market of GEO_MARKETS) {
    const path = `maerkte/${market.slug}`;
    for (const locale of routing.locales) {
      sitemapEntries.push({
        url: getLocalizedUrl(locale, path),
        lastModified: new Date('2026-06-14T15:00:00Z'),
        changeFrequency: 'weekly',
        priority: 0.7,
        alternates: {
          languages: {
            de: getLocalizedUrl('de', path),
            en: getLocalizedUrl('en', path),
            ar: getLocalizedUrl('ar', path),
            'x-default': getLocalizedUrl('de', path),
          },
        },
      });
    }
  }

  return sitemapEntries;
}
```

### Proposed `app/robots.ts`

```ts
import type { MetadataRoute } from 'next';

export const dynamic = 'force-static';

export default function robots(): MetadataRoute.Robots {
  const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || 'https://k-aqua.de';
  return {
    rules: {
      userAgent: '*',
      allow: '/',
    },
    sitemap: `${siteUrl}/sitemap.xml`,
    host: siteUrl,
  };
}
```

## 5. Verification Method

To independently verify the proposed sitemap and robots implementation after deployment:

1. **Verify Sitemap XML Format**:
   Run `curl -I http://localhost:3001/sitemap.xml` and inspect that the response returns `200 OK` status and `Content-Type: application/xml`.
2. **Count XML URLs**:
   Use standard CLI tools to extract and count sitemap entries:
   ```bash
   curl -s http://localhost:3001/sitemap.xml | grep -o '<loc>' | wc -l
   ```
   *Expectation:* Exactly `135` entries are outputted.
3. **Verify Hreflang Alternates**:
   Check if the alternate links are present in the XML sitemap for any arbitrary route (e.g. `/de/loesungen`):
   Each `<url>` tag must contain:
   ```xml
   <xhtml:link rel="alternate" hreflang="de" href="https://k-aqua.de/de/loesungen" />
   <xhtml:link rel="alternate" hreflang="en" href="https://k-aqua.de/en/loesungen" />
   <xhtml:link rel="alternate" hreflang="ar" href="https://k-aqua.de/ar/loesungen" />
   <xhtml:link rel="alternate" hreflang="x-default" href="https://k-aqua.de/de/loesungen" />
   ```
4. **Verify Robots.txt directives**:
   Run `curl -s http://localhost:3001/robots.txt` and verify that the output matches:
   ```txt
   User-agent: *
   Allow: /
   Sitemap: https://k-aqua.de/sitemap.xml
   Host: https://k-aqua.de
   ```
5. **Verify Next.js build passes**:
   Run `pnpm build` and verify that both static files `/sitemap.xml` and `/robots.txt` are created successfully without any typescript or layout compilation errors.
