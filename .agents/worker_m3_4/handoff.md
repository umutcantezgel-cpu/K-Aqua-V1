## 2. Logic Chain

- **Coping with Missing GLB/KTX2 Assets**: Since the models `/models/k-aqua-product.glb` and `/textures/water_normals.ktx2` are not present in the workspace, placing `useGLTF` directly inside standard components would suspend and throw 404 network errors, crashing the canvas. Therefore, I wrote a `GLTFErrorBoundary` React Class Component to catch promise rejection errors thrown by Drei's loader and gracefully fall back to a high-poly procedural 3D model container (`ProceduralFallback`).
- **Dynamic Shading & WebGL 2.0**: The procedural fallback utilizes an custom-instantiated `WaterShaderMaterial` subclass generated by `shaderMaterial` that sets the `glslVersion` to `THREE.GLSL3` (WebGL 2.0 / GLSL 300 ES compliant). Inside the shader, vertex displacement is calculated using a 3D simplex noise fractional Brownian motion (fBm) over time. The fragment shader incorporates dynamic Fresnel refraction at the pipe edges, reflections mapped via `samplerCube`, and custom fake caustics.
- **Garbage Collection Optimization**: Inside `WaterBubbles.tsx`, which simulates 10,000 bubbles using `THREE.InstancedMesh`, all temporary math operators (like `THREE.Object3D` matrices) are initialized once in `useMemo` hooks. In the `useFrame` render loop, we iterate over the 10,000 pre-computed bubble parameters, set the matrix on the dummy object, and copy it directly to the instances matrix buffer. No new instances of `THREE.Vector3` or `THREE.Matrix4` are generated inside `useFrame`, keeping heap allocations at absolute zero.
- **Cinematic Transition & Raycasting**: Bounding volume trees are computed on load/mount on all buffer geometries (both GLB-loaded and procedural fallback) using `three-mesh-bvh` prototype extension, speeding up pointer raycasts. Smooth orbital target tracking and camera transitions are calculated via GSAP tweens inside the `<CameraController />` component. Subcomponent meshes (such as flanges, branches, and handles) animate outward via GSAP when `isExploded` is active.

## 3. Caveats

- Since no actual environment map `.hdr` file or cube textures are loaded in standard transparent fallback mode, environment reflection inside the shader is disabled by default via `uUseEnvMap = 0.0`. It is fully functional once a cube-mapped texture or scene environment is mapped onto the custom material's `uEnvironmentMap` uniform.
- Post-processing effects stack requires a valid WebGL2 context to render SSAO correctly. Standard setups support this. In case of hardware/WebGL limitations on older mobile devices, postprocessing effects degrade gracefully.
