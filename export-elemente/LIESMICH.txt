K-AQUA ELEMENT-BIBLIOTHEK — EXPORT
==================================

Alle Elemente aus dem K-Aqua Element-Archiv, exportiert als GLOBALE
Bibliothek + einzelne Markup-Bausteine. Kernprinzip:

  CSS und JS werden EINMAL pro Seite eingebunden.
  Markup kann danach BELIEBIG OFT eingesetzt werden — auf jeder Seite,
  mehrfach pro Seite, ohne dass Styles oder Skripte doppelt geladen werden.
  Nichts "verschwindet" beim Seitenwechsel: die Bibliothek gehört ins
  globale Layout (Head/Footer), die Markup-Bausteine in die Seiten.

DATEIEN
-------
kaqua-elemente.css   Gesamtes Element-CSS inkl. Token-Basis (Light +
                     OLED-Dark via data-theme am <html>). Alle Regeln
                     tragen Hex-Fallbacks (Light) und funktionieren auch
                     ohne eigenes Token-Setup.
kaqua-elemente.js    Initialisiert alle interaktiven Elemente (Canvas-FX,
                     Menüs, Reveals, Stepper, CountUp …). Mehrfach-Aufruf
                     sicher (Guard: data-ka-init). Für dynamisch nach-
                     geladenes Markup: window.KAquaElemente.init()
markup/              Ein HTML-Baustein pro Element, nummeriert nach
                     Segment — in dieser Reihenfolge nacheinander
                     integrierbar (01 → 11). Jeder Baustein enthält die
                     KI-Notiz als Kommentar.
KI-PROMPT.txt        Übergabetext für eine KI, die die Integration in die
                     bestehende Website ausführt.

EINBINDUNG (einmal, global)
---------------------------
  <html data-theme="light">   (oder "dark" — OLED)
  <head>
    <link rel="stylesheet" href="kaqua-elemente.css">
  </head>
  <body>
    … Markup-Bausteine …
    <script src="kaqua-elemente.js" defer></script>
  </body>

Schriften: Outfit + Inter — entweder lokal (fonts/outfit-variable-latin.woff2,
fonts/inter-variable-latin.woff2 via @font-face) oder Google Fonts:
@import url('https://fonts.googleapis.com/css2?family=Outfit:wght@100..900&family=Inter:wght@100..900&display=swap');

SEQUENZIELLE INTEGRATION
------------------------
Empfohlene Reihenfolge = Ordner-Nummern:
  01 Tokens (Basis, zuerst) → 02 Motive → 03 UI → 04 Dekor → 05 Sections
  → 06 Layout → 07 Trenner → 08-11 FX (nach Bedarf)
Jeder Schritt ist in sich geschlossen: Baustein-Markup einsetzen, fertig —
CSS/JS sind bereits global da. Pro Element gilt die KI-Notiz im Kommentar
(Einsatzort, Grenzen, Konfiguration über data-Attribute).

WIEDERVERWENDUNG
----------------
- Gleiche Klasse mehrfach pro Seite: erlaubt (JS initialisiert jede Instanz).
- Einzige Ausnahme: IDs in SVG-Defs (cert-ring: ka-cert-orbit-1) müssen pro
  Instanz eindeutig sein — bei Mehrfachnutzung ID + href anpassen.
- Alle Farben über semantische Tokens: Anpassung zentral in kaqua-elemente.css
  (Abschnitt "01 · css-variables"), nie im Markup.
